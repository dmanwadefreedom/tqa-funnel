## DM Sequence (3 Messages Over 2 Weeks)

### Message 1: Connection Request Note

**When:** Sent with the connection request
**Character limit:** 300 characters
**Goal:** Get the connection accepted. Nothing else. No pitch. No link.

**Template (289 chars):**
```
Hi [Name] — I'm DJ Naronikar, 28-year CISO (Deloitte, EY, Shell). I work with IT professionals exploring cybersecurity careers. Noticed your background in [their field]. Happy to connect.
```

**Personalization rules:**
- `[Name]` = first name only
- `[their field]` = pulled from their headline. Examples: "cloud infrastructure," "network administration," "systems engineering," "database management"
- If their headline is generic ("IT Professional at Company"), use: "Noticed your experience in IT infrastructure."

**What NOT to do:**
- Don't mention TQA
- Don't include a link
- Don't ask a question
- Don't pitch anything
- Don't use exclamation marks

---

### Message 2: Value First

**When:** 3 days after connection is accepted
**Character limit:** No hard limit (LinkedIn DM), but keep under 500 characters
**Goal:** Start a conversation. Deliver value. Plant the idea.

**Template (487 chars):**
```
Thanks for connecting, [Name].

Quick question — have you considered cybersecurity as a career path?

With your [their field] background, the transition is more structured than most people think. The skills transfer directly — it's more about positioning and knowing the landscape than starting over.

I put together a breakdown on this:
[link to YouTube video or LinkedIn article]

No pitch — just thought it might be relevant given your background.
```

**Link options (choose one per prospect):**
- YouTube video: "How IT Professionals Break Into Cybersecurity" (DJ needs to publish this)
- LinkedIn article: "The IT-to-Cybersecurity Path Nobody Talks About" (DJ needs to publish this)
- LinkedIn post: Any high-performing post about career transition

**Personalization rules:**
- `[Name]` = first name
- `[their field]` = same as Message 1. Keep consistent.
- The link should be the SAME piece of content for all prospects in a given week (easier to track engagement)

**What NOT to do:**
- Don't mention the program
- Don't mention price
- Don't use "limited spots" or urgency language
- Don't follow up if they don't respond (yet — that's what Message 3 timing handles)

---

### Message 3: Soft CTA

**When:** 7 days after Message 2, ONLY if they engaged (replied, clicked the link, liked the content, or viewed DJ's profile)
**Character limit:** Keep under 600 characters
**Goal:** Invite to the live briefing. One CTA. Low pressure.

**Template (493 chars):**
```
Hey [Name] — glad the [video/article] was useful.

I'm running a live career briefing on [date] — 90 minutes on the cybersecurity career path for IT professionals.

We cover:
- The actual transition roadmap (not just "get certified")
- What hiring managers look for
- A live LinkedIn profile audit

$27 to attend. Here are the details: [link to briefing page]

No pressure. If the timing doesn't work, I do these monthly.
```

**Personalization rules:**
- `[Name]` = first name
- `[date]` = next briefing date (e.g., "April 11")
- `[video/article]` = reference whichever content you shared in Message 2
- `[link to briefing page]` = thequantumachievers.com/briefing (or wherever the briefing registration lives)

**What NOT to do:**
- NEVER send Message 3 to someone who didn't engage with Message 2. They go to the cold list.
- Don't discount the $27. It's already low.
- Don't say "only $27" — just state the price without minimizing it.
- Don't send to people who ignored Message 2 and also didn't view DJ's profile.

---

### If They Reply at Any Stage

**If positive/curious:** Respond conversationally. Answer their question. Don't pitch unless they ask about the program. If they ask, share the briefing link.

**If they ask "what's the program?":** Brief answer: "It's a 90-day structured cybersecurity career transition program I run. The best way to see if it's a fit is the live briefing — [link]. Happy to answer any specific questions here too."

**If negative/not interested:** "No worries at all. Best of luck with everything, [Name]." — Do not follow up again.

**If no response to Message 2 AND no profile view:** Tag as cold. Do not send Message 3. Move to long-term nurture (see GHL workflow below).

## GHL Workflow Spec (For Agresh to Build)

### Workflow Name: `TQA LinkedIn DM Funnel`

### Overview

```
LinkedIn Connection Accepted
        ↓
   [GHL Contact Created]
   Tag: tqa-linkedin-dm-funnel
        ↓
   Wait 3 days
        ↓
   [Send Message 2 — Value]
   (manual send via LinkedIn, triggered by GHL task notification)
        ↓
   Wait 7 days
        ↓
   [Check Engagement]
        ↓
   ┌─── Engaged? ───┐
   ↓ YES             ↓ NO
   Send Message 3    Tag: tqa-linkedin-cold
   Tag: tqa-engaged  Move to Long-Term Nurture
        ↓
   [Check Registration]
        ↓
   ┌─── Registered? ───┐
   ↓ YES                ↓ NO
   Tag: tqa-briefing    Tag: tqa-engaged-no-briefing
   Enter pre-class       Wait 14 days → re-invite
   email sequence        to next briefing (once)
```

### Step-by-Step GHL Build

#### Step 1: Contact Creation

**Trigger:** Manual entry or CSV import (LinkedIn doesn't have a direct API to GHL — DJ or Agresh manually logs accepted connections)

**Alternative trigger (if using a LinkedIn automation tool like Dripify or Expandi):** Webhook from the tool on "connection accepted" event. But be careful — LinkedIn automation tools carry account risk. Manual is safer.

**Contact fields to capture:**
| Field | Source |
|-------|--------|
| First name | LinkedIn profile |
| Last name | LinkedIn profile |
| Email | Only if visible on LinkedIn (often not) |
| LinkedIn URL | Profile URL |
| Current title | From their headline |
| Current company | From their profile |
| Metro area | From their location |
| Source | "LinkedIn DM Funnel" |

**Tags applied on creation:**
- `tqa-linkedin-dm-funnel`
- `tqa-msg1-sent` (connection request was already sent)

---

#### Step 2: Wait 3 Days

**GHL action:** Wait step — 3 calendar days

---

#### Step 3: Message 2 Notification

**GHL action:** Internal notification (email or SMS to DJ/Agresh) with:
- Contact name
- LinkedIn URL
- Their current title/field
- The Message 2 template pre-filled with personalization

**Why a notification instead of auto-send:** LinkedIn DMs cannot be sent programmatically through GHL without a third-party tool. DJ sends manually from the notification prompt. This keeps it authentic and avoids LinkedIn account flags.

**After sending:** Agresh (or DJ) updates the contact in GHL:
- Add tag: `tqa-msg2-sent`
- Add note: "Message 2 sent [date]"

---

#### Step 4: Wait 7 Days

**GHL action:** Wait step — 7 calendar days after Message 2 is marked as sent

---

#### Step 5: Engagement Check

**GHL action:** Internal notification to DJ/Agresh to check engagement.

**What counts as "engaged":**
- They replied to Message 2 (any reply)
- They clicked the link in Message 2 (check YouTube analytics or LinkedIn article views if possible)
- They viewed DJ's LinkedIn profile after receiving Message 2
- They liked or commented on DJ's recent LinkedIn posts

**Agresh checks and updates GHL:**

**If engaged:**
- Add tag: `tqa-linkedin-engaged`
- Remove tag: `tqa-msg2-sent`
- Trigger: Message 3 notification

**If NOT engaged:**
- Add tag: `tqa-linkedin-cold`
- Remove tag: `tqa-msg2-sent`
- Move to: Long-term nurture pipeline

---

#### Step 6A: Message 3 (Engaged Path)

**GHL action:** Internal notification with Message 3 template pre-filled.

DJ sends manually via LinkedIn.

**After sending:** Update contact:
- Add tag: `tqa-msg3-sent`
- Add note: "Message 3 sent [date] — briefing invite for [date]"

---

#### Step 6B: Cold Path

**GHL action:** Move contact to pipeline: `TQA Long-Term Nurture`

**Long-term nurture actions:**
- If DJ publishes a particularly strong piece of content (viral LinkedIn post, major podcast episode), Agresh can send a one-off "thought you'd find this interesting" DM to the cold list. Maximum once per month.
- If cold contacts engage with DJ's public content later (like, comment, share), Agresh moves them back to the engaged pipeline and sends Message 3.

---

#### Step 7: Registration Check (After Message 3)

**GHL action:** Wait 5 days after Message 3 sent, then check.

**If registered for briefing:**
- Add tag: `tqa-briefing-registrant`
- Remove tags: `tqa-msg3-sent`, `tqa-linkedin-engaged`
- Enter workflow: `TQA Pre-Briefing Email Sequence` (separate workflow — reminder emails before the briefing)

**If NOT registered but still engaged (replied positively but didn't register):**
- Add tag: `tqa-engaged-no-briefing`
- Wait 14 days → Send one re-invite to the next monthly briefing
- If still no registration → move to Long-Term Nurture

**If no response to Message 3:**
- Add tag: `tqa-linkedin-cold`
- Move to Long-Term Nurture

## Risk Management

| Risk | Mitigation |
|------|------------|
| LinkedIn account restriction (too many connection requests) | Stay under 25/day. Never use automation tools. Personalize every note. |
| Low acceptance rate (<20%) | Refine targeting. Test different connection note wording. Ensure DJ's profile is fully updated (see TQA-UNIFIED-BIOS.md). |
| Low engagement after Message 2 (<10%) | Test different value content (video vs article vs post). Test different timing (3 days vs 5 days). |
| Prospects feel pitched | Message sequence is designed to avoid this. If someone says "don't sell me," respond graciously and tag as cold. Never push. |
| GHL-LinkedIn integration limitations | This workflow is semi-manual by design. GHL handles contact management and notifications. LinkedIn activity is done by DJ manually. This is intentional — it keeps the outreach authentic and avoids platform risk. |
| Sales Navigator subscription cost | $99/mo. At even one $27 briefing conversion per week, it pays for itself in 1 week. The real ROI is in TQA program applications that come from briefing attendees. |
